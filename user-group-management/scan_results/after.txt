=== TECHNICAL NOTES ===

1. PASS_MIN_LEN not present in /etc/login.defs on RHEL 9
   RHEL 9 enforces password complexity and minimum length through
   PAM/pwquality (/etc/security/pwquality.conf) rather than login.defs.
   The minlen parameter in pwquality.conf controls minimum length.
   This is a RHEL 9 architectural difference from earlier RHEL versions.

2. login.defs affects NEW accounts only
   Changes to /etc/login.defs apply to accounts created after the change.
   Existing accounts require per-account remediation via chage.
   This is why audit_accounts.sh checks both system defaults AND
   per-account chage values — system-level compliance does not
   guarantee per-account compliance.

3. Epoch date gotcha on existing accounts
   Applying maxdays to accounts with "Last password change: never"
   causes immediate password expiration (60 days from epoch = 1970).
   Always run chage -l before applying aging to existing accounts.
   Fix: chage -d $(date +%Y-%m-%d) <username> before applying maxdays.
