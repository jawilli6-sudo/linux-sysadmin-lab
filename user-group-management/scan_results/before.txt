=== TECHNICAL NOTE ===
jbeko's "Last password change" was set to "never" (epoch day 0).
When PASS_MAX_DAYS 60 was applied, PAM calculated the password had
expired 60 days after epoch (January 1, 1970) — decades ago.
This locked sudo access immediately after chage was applied.

Resolution: Root login via VirtualBox console used to run:
  chage -d $(date +%Y-%m-%d) jbeko
  passwd jbeko
This reset the last change date to today before re-applying aging controls.

This is a known operational gotcha when applying password aging to
existing accounts where the last change date was never initialized.
Senior sysadmins always check chage -l before applying maxdays to
accounts with "Last password change: never."
